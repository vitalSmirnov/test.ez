## 1 Клиентская часть состоит из vue + pinia + quasar + bun

## 2 для сборки стоит запустить `bun run build` - это сгенерирует статику для сервера и можно будет использовать один инстанс сервера на порту [3000](http://localhost:3000)

## 3 Так же можно запустить локально `bun run dev` - CORS предусмотрены благодаря vite.config.ts и его proxy для сервера, что позволяет использовать как отдельный инстанс на порту [5173](http://localhost:5173) без переписывания CORS

### Структура

в src есть две папки:

- components - для компонентов vue
- shared - для хранилища pinia и api handlers

## ВАЖНО
bun полностью совместим с npm, поэтому можно использовать классические команды, вместо bun
